# Gravesown: After the Silence

Мрачный total-conversion мод для Minecraft Java 1.21.1 на NeoForge.

Природа исчезла после события, которое выжившие называют Тишиной. Мир
воспроизводит новую экосистему из костей, падали и земли. Это самостоятельный
проект: он не копирует существ, лор, код или ресурсы других модов.

## Самый простой запуск

1. Дважды нажми setup.cmd. Скрипт скачает локальную Java 21 и подготовит проект.
2. Дважды нажми test.cmd. Серверный автотест сам запустится, проверит Пустопаса и закроется.
3. Дважды нажми play.cmd.
4. В тестовом Minecraft создай отдельный Creative-мир с читами.

Системная Java и глобальный Gradle не нужны.

Первая игровая альфа уже содержит Пустопаса, его модель, текстуру, яйцо призыва,
AI и серверную замену естественных ванильных спавнов в обычном мире. Быстрый
ручной вызов:

```mcfunction
/summon gravesown:hollow_grazer ~ ~ ~
```

Для проверки поведения используй `/time set day`, `/time set night` и сравни
реакцию существа на здорового и тяжело раненого игрока.

## Кнопки в корне

| Файл | Назначение |
|---|---|
| setup.cmd | Один раз подготавливает Java 21 и зависимости |
| test.cmd | Запускает автоматические серверные GameTests и сам закрывается |
| play.cmd | Запускает тестовый клиент с модом |
| build.cmd | Проверяет проект и кладёт готовый JAR в dist |
| doctor.cmd | Диагностирует окружение и компиляцию |
| server.cmd | Запускает локальный dedicated server |
| diagnostics.cmd | Собирает безопасный ZIP с логами для исправления ошибки |

## Новый чат с ИИ

Открой новый чат прямо в этой папке и напиши:

> Продолжай разработку. Сначала полностью прочитай AGENTS.md и документы,
> указанные в нём. Проверь проект через doctor.cmd и выполни Next action из
> docs/STATUS.md.

Готовый текст также лежит в prompts/CONTINUE_IN_NEW_CHAT.md.

## Техническая база

- Minecraft Java Edition 1.21.1
- NeoForge 21.1.235
- Java 21
- ModDevGradle 2.0.141
- Gradle Wrapper 9.2.1
- mod id: gravesown
- package: dev.gravesown

Версии закреплены. Не обновляй их случайно.

## Где искать результат и ошибки

- релизный JAR: dist/
- результаты Gradle: build/
- клиентский лог: run/logs/latest.log
- crash reports: run/crash-reports/
- состояние проекта для нового ИИ: docs/STATUS.md

Подробное тестирование описано в docs/TESTING.md.
