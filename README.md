# Gravesown: After the Silence

Оригинальный total-conversion мод для Minecraft Java Edition 1.21.1 на
NeoForge. Игрок переживает крушение на дикой низкотехнологичной планете,
изучает местную экосистему и постепенно переходит от примитивного выживания к
собственным производственным, энергетическим и автоматизированным системам.

Проект не копирует существ, лор, код, модели или ресурсы других модов.

## Скачать и запустить

На Windows:

1. Скачай исходники через Git или распакуй GitHub Release.
2. Дважды нажми `launcher.cmd`.
3. При первом запуске автоматически откроется `setup.cmd`: он подготовит Java
   21 и официальные зависимости Minecraft 1.21.1/NeoForge.
4. После подготовки откроется лаунчер Gravesown. Нажми **Играть**.

Повторные запуски используют уже подготовленный внешний кэш и не скачивают
Minecraft заново. Для запуска без графического лаунчера можно нажать
`play.cmd`; он использует тот же автоматический bootstrap.

Minecraft и его модифицированные игровые файлы не хранятся в репозитории и не
распространяются в релизном архиве. Первый запуск получает необходимые
официальные зависимости через настроенную NeoForge/Gradle-среду.

## Что уже есть в альфе

- девять собственных биомов и полностью заменённая поверхность мира;
- 23 вида местной фауны, водная экосистема и транспорт;
- Gravework 4x4, печь, лесопилка, полевая кухня и расширенный ящик;
- ранние инструменты, броня, метательное копьё, фермерство и приготовление еды;
- Кодекс выжившего с сюжетом, рецептами, цепочками создания и гайдом;
- могилы, руины и редкие биомные лагеря;
- русская и английская локализация;
- Windows-лаунчер и серверно-авторитетная логика для одиночной игры и
  dedicated server.

## Где хранятся Java, кэши и миры

Репозиторий остаётся небольшим: загруженная Java, рабочие клиенты, серверы,
логи и тестовые миры находятся вне проекта:

```text
%LOCALAPPDATA%\Gravesown\
  runtime\          Java 21
  cache\projects\   отдельный кэш Gradle для каждой рабочей копии
  runs\client\      обычный клиент и его сохранения
  runs\server\      dedicated server
  runs\tests\       одноразовые тестовые миры
  state\            отметка завершённой установки
```

Общий кэш зависимостей Gradle находится в `%USERPROFILE%\.gradle`. Эти папки
не попадают в Git и не раздувают рабочую копию. `clean-storage.cmd` удаляет
только безопасные одноразовые данные разработки и не трогает игровые
сохранения.

## Основные команды

| Файл | Назначение |
|---|---|
| `launcher.cmd` | Подготавливает проект при необходимости и открывает Windows-лаунчер |
| `play.cmd` | Подготавливает проект при необходимости и запускает клиент напрямую |
| `setup.cmd` | Единственная штатная команда, которой разрешено скачивать зависимости |
| `doctor.cmd` | Проверяет Java, версии, внешний кэш и офлайн-компиляцию |
| `test.cmd` | Запускает серверные NeoForge GameTests |
| `worldtest.cmd` | Проверяет реальные сгенерированные чанки |
| `clienttest.cmd` | Выполняет финальный клиентский smoke-тест в 1920x1080 |
| `verify-all.cmd` | Запускает основной автоматический набор проверок |
| `build.cmd` | Собирает релизный JAR в `dist/` |
| `gradle-offline.cmd` | Безопасно запускает произвольную Gradle-задачу офлайн |
| `github-ready.cmd` | Проверяет сборку, лаунчер и безопасность содержимого Git |
| `package-release.cmd` | Создаёт JAR и готовый Windows ZIP для GitHub Releases |
| `clean-storage.cmd` | Чистит одноразовые кэши и тестовые миры |
| `server.cmd` | Запускает локальный dedicated server |
| `diagnostics.cmd` | Собирает безопасный ZIP с диагностикой |

Пример ручной команды:

```bat
gradle-offline.cmd compileJava
```

## Подготовка GitHub-релиза

Перед коммитом:

```bat
github-ready.cmd
```

Для создания публикуемых файлов:

```bat
package-release.cmd
```

Результаты:

- `dist/gravesown-<version>.jar`;
- `release/Gravesown-After-the-Silence-<version>-Windows.zip`.

ZIP содержит исходники, готовый лаунчер и собранный мод. Пользователь
распаковывает его и запускает `launcher.cmd`. Крупные игровые зависимости
устанавливаются только при первом запуске и остаются вне распакованной папки.

GitHub Actions в `.github/workflows/` проверяют Windows-сборку и создают те же
артефакты. Workflow публикации запускается тегом вида `v0.1.0-alpha.1`.

## Новый чат с ИИ

Открой новый чат в корне проекта и напиши:

> Продолжай разработку. Полностью прочитай AGENTS.md и документы, указанные в
> нём. Запусти doctor.cmd и выполни ровно Next action из docs/STATUS.md.

Готовый текст лежит в `prompts/CONTINUE_IN_NEW_CHAT.md`. Источники истины:
`docs/STATUS.md`, `docs/DESIGN.md`, `docs/ART_DIRECTION.md`,
`docs/TOTAL_CONVERSION_PLAN.md`, `docs/ROADMAP.md` и `docs/DECISIONS.md`.
